## 系统概述

OpenCode 项目实现了一个灵活且可扩展的模型提供商系统，支持多种 AI 模型提供商（如 OpenAI、Anthropic、Google 等）。该系统提供了统一的 API 接口，允许用户通过单一接口访问不同的 AI 模型服务。

### 核心能力
- 支持多种模型提供商
- 提供商故障转移机制
- 粘性提供商选择
- 多提供商负载均衡
- 内置用户计费系统
- Bring Your Own Key (BYOK) 支持
- 请求/响应格式转换

### 3. 模型配置结构

**文件位置**: [packages/console/core/src/model.ts](file:///workspace/opencode/packages/console/core/src/model.ts)

```typescript
const ModelSchema = z.object({
  name: z.string(),
  cost: ModelCostSchema,
  cost200K: ModelCostSchema.optional(),
  allowAnonymous: z.boolean().optional(),
  byokProvider: z.enum(["openai", "anthropic", "google"]).optional(),
  stickyProvider: z.enum(["strict", "prefer"]).optional(),
  trialProviders: z.array(z.string()).optional(),
  trialEnded: z.boolean().optional(),
  fallbackProvider: z.string().optional(),
  rateLimit: z.number().optional(),
  providers: z.array(
    z.object({
      id: z.string(),
      model: z.string(),
      weight: z.number().optional(),
      disabled: z.boolean().optional(),
      storeModel: z.string().optional(),
      payloadModifier: z.record(z.string(), z.any()).optional(),
      safetyIdentifier: z.boolean().optional(),
    }),
  ),
})
```

**关键字段说明**:
- `byokProvider`: 指定用于 BYOK（自带密钥）的提供商
- `stickyProvider`: 粘性提供商策略
  - `strict`: 严格使用同一提供商，不允许故障转移
  - `prefer`: 优先使用，但允许故障转移
- `fallbackProvider`: 故障转移时使用的后备提供商
- `providers`: 该模型支持的提供商列表，每个提供商包含：
  - `id`: 提供商 ID
  - `model`: 提供商侧的模型名称
  - `weight`: 权重（用于负载均衡）
  - `disabled`: 是否禁用

**核心文件**: [packages/console/app/src/routes/zen/util/handler.ts](file:///workspace/opencode/packages/console/app/src/routes/zen/util/handler.ts)

### 完整处理流程

```
1. 接收请求
   ↓
2. 解析 API Key & 认证
   ↓
3. 验证模型可用性
   ↓
4. 初始化限制器 (速率限制、试用限制)
   ↓
5. 选择提供商
   ├─ 检查 BYOK (自带密钥)
   ├─ 检查粘性提供商
   ├─ 试用提供商随机选择
   ├─ 权重负载均衡
   └─ 故障转移回退
   ↓
6. 验证计费状态
   ↓
7. 转换请求格式
   ↓
8. 发送请求到提供商 (带 429 重试)
   ↓
9. 处理响应 (流式/非流式)
   ↓
10. 转换响应格式
    ↓
11. 记录使用量 & 更新计费
    ↓
12. 返回响应
```

#### 2. 提供商选择阶段

```typescript
function selectProvider(/* ... */) {
  const modelProvider = (() => {
    // 优先检查 BYOK
    if (authInfo?.provider?.credentials) {
      return modelInfo.providers.find((provider) => provider.id === modelInfo.byokProvider)
    }

    // 检查粘性提供商
    if (stickyProvider) {
      const provider = modelInfo.providers.find((provider) => provider.id === stickyProvider)
      if (provider) return provider
    }

    // 试用提供商
    if (trialProviders) {
      const trialProvider = trialProviders[Math.floor(Math.random() * trialProviders.length)]
      const provider = modelInfo.providers.find((provider) => provider.id === trialProvider)
      if (provider) return provider
    }

    // 普通选择：基于权重
    if (retry.retryCount !== MAX_FAILOVER_RETRIES) {
      const providers = modelInfo.providers
        .filter((provider) => !provider.disabled)
        .filter((provider) => !retry.excludeProviders.includes(provider.id))
        .flatMap((provider) => Array<typeof provider>(provider.weight ?? 1).fill(provider))

      // 使用 session ID 的后 4 个字符选择提供商
      const identifier = sessionId.length ? sessionId : ip
      let h = 0
      const l = identifier.length
      for (let i = l - 4; i < l; i++) {
        h = (h * 31 + identifier.charCodeAt(i)) | 0
      }
      const index = (h >>> 0) % providers.length
      const provider = providers[index || 0]
      if (provider) return provider
    }

    // 回退提供商
    return modelInfo.providers.find((provider) => provider.id === modelInfo.fallbackProvider)
  })()

  // ... 返回完整的提供商信息
}
```

#### 3. 故障转移机制

```typescript
const retriableRequest = async (retry: RetryOptions = { excludeProviders: [], retryCount: 0 }) => {
  const providerInfo = selectProvider(/* ... */)
  // ... 发送请求

  if (
    res.status !== 200 &&
    res.status !== 404 &&
    modelInfo.stickyProvider !== "strict" &&
    modelInfo.fallbackProvider &&
    providerInfo.id !== modelInfo.fallbackProvider
  ) {
    return retriableRequest({
      excludeProviders: [...retry.excludeProviders, providerInfo.id],
      retryCount: retry.retryCount + 1,
    })
  }

  return { providerInfo, reqBody, res, startTimestamp }
}
```

#### 4. 429 重试

```typescript
async function fetchWith429Retry(url: string, options: RequestInit, retry = { count: 0 }) {
  const res = await fetch(url, options)
  if (res.status === 429 && retry.count < MAX_429_RETRIES) {
    await new Promise((resolve) => setTimeout(resolve, Math.pow(2, retry.count) * 500))
    return fetchWith429Retry(url, options, { count: retry.count + 1 })
  }
  return res
}
```

## 关键特性

### 1. 粘性提供商 (Sticky Provider)

**文件位置**: [packages/console/app/src/routes/zen/util/stickyProviderTracker.ts](file:///workspace/opencode/packages/console/app/src/routes/zen/util/stickyProviderTracker.ts)

```typescript
export function createStickyTracker(stickyProvider: "strict" | "prefer" | undefined, session: string) {
  if (!stickyProvider) return
  if (!session) return
  const key = `sticky:${session}`

  return {
    get: async () => {
      return await Resource.GatewayKv.get(key)
    },
    set: async (providerId: string) => {
      await Resource.GatewayKv.put(key, providerId, { expirationTtl: 86400 })
    },
  }
}
```

**工作原理**：
- 使用 KV 存储存储 session 到提供商的映射
- 过期时间 24 小时
- 同一会话优先使用同一提供商

### 2. Bring Your Own Key (BYOK)

用户可以配置自己的 API Key，系统会优先使用该 Key 调用指定的提供商：

```typescript
if (authInfo?.provider?.credentials) {
  return modelInfo.providers.find((provider) => provider.id === modelInfo.byokProvider)
}
```

### 3. 权重负载均衡

每个提供商可以配置权重，系统根据权重进行负载均衡：

```typescript
const providers = modelInfo.providers
  .filter((provider) => !provider.disabled)
  .flatMap((provider) => Array<typeof provider>(provider.weight ?? 1).fill(provider))
```

### 4. 故障转移

当提供商返回非 200 状态码（除了 404）且不是严格粘性模式时，系统会自动尝试其他提供商：

```typescript
if (
  res.status !== 200 &&
  res.status !== 404 &&
  modelInfo.stickyProvider !== "strict" &&
  modelInfo.fallbackProvider &&
  providerInfo.id !== modelInfo.fallbackProvider
) {
  return retriableRequest({
    excludeProviders: [...retry.excludeProviders, providerInfo.id],
    retryCount: retry.retryCount + 1,

}
```

### 5. 格式转换

系统支持在不同 API 格式之间进行转换，使客户端可以使用一种格式调用所有提供商：

```

    ↓

    ↓

    ↓
```

OpenCode 的模型提供商系统是一个设计完善的多提供商管理系统，具有以下优势：

1. **高度可扩展**：轻松添加新的提供商
